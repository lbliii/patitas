# 0.4.0

**Released**: 2026-06-15

A security-and-correctness release that also introduces content linting. Users on
0.3.5 are encouraged to upgrade: this release closes an XSS sanitizer bypass, a
denial-of-service crash on nested input, and an end-to-end directive-rendering
regression.

---

## Security & Correctness Fixes

- **Sanitizer URL-scheme bypass (XSS).** Entity-encoded (`&#106;avascript:`),
  mixed-case (`JavaScript:`), and whitespace/control-obfuscated (`java\tscript:`)
  schemes slipped through `strip_dangerous_urls`, `llm_safe`, `web_safe`, and
  `strict`. URLs are now HTML-unescaped and control/whitespace-stripped before the
  scheme check, and the pre-built policies use an allow-list (`https`/`http`/`mailto`
  plus relative URLs) instead of a fixed deny-list.

- **Uncaught `RecursionError` on deeply nested input (DoS).** Deeply nested block
  quotes / lists / directives crashed the interpreter on sub-1KB input. Nesting is
  now bounded by the new `max_nesting_depth` (default 100) and raises a catchable
  `ParseError`.

- **Directive rendering was broken end-to-end.** `HtmlRenderer._render_directive`
  invoked handlers with the wrong arity and swallowed the resulting `TypeError`, so
  every directive — all five builtins (`note`/admonitions, `tab-set`/`tab-item`,
  `dropdown`, `container`) and any custom `@directive` — silently degraded to a
  generic `<div>`. Handlers are now invoked correctly and handler exceptions
  propagate.

- **`limit_depth()` was a documented no-op.** It now actually prunes block content
  nested beyond the limit.

---

## New: Content Linting

- **`lint(source)` and a `LintRule` protocol** that inspect the typed AST plus raw
  source, emitting `Diagnostic`s that reuse `node.location`. `lint` accepts a
  Markdown string or a pre-parsed `Document` (via `text=`). Three starter rules
  ship: `heading-increment`, `no-empty-link`, and `trailing-whitespace`.
- `Diagnostic`, `LintRule`, and `Severity` (with `.to_lsp()`) are exported from
  `patitas.linting` and the `patitas` top level; the richer surface (`Linter`,
  `LintContext`, `LintRuleRegistry`/`Builder`, `create_default_lint_registry`, and
  the rule classes) lives in `patitas.linting`. See `docs/linting.md`.

---

## Other Changes

- **`directive` decorator is now a public export** (`from patitas import directive`).
- **`max_nesting_depth`** — new `ParseConfig` field and `Markdown(...)` argument.
- **Behavioral test suites** added for directive rendering, adversarial input,
  autolinks, and sanitize policy composition — asserting real rendered output
  through the public API.
- **Docs corrected to match measured reality** — README dependency/perf claims,
  ROADMAP perf gap, `docs/security.md` (default `render()` does **not** sanitize),
  migration guide, and GFM-compliance notes.

## Installation

```bash
pip install patitas>=0.4.0
pip install patitas[syntax]   # Optional: syntax highlighting via Rosettes
```

---

## Breaking Changes

None — backwards compatible. Deeply nested input that previously crashed with an
uncaught `RecursionError` now raises a catchable `ParseError` once it exceeds
`max_nesting_depth` (default 100); raise the limit if you intentionally parse
deeper structures.
