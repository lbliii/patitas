# 0.3.5

**Released**: 2026-03-09

Performance improvements, benchmark suite, and fixes for excerpt and directive parsing.

---

## Highlights

- **`_line_start_for_offset()`** — O(log n) line boundary lookup with lazy line index. Replaces repeated `rfind("\n", 0, offset)` (O(n) per call) with bisect-based lookup. Used by directive `preserves_raw_content`, list parsing, and block core.

- **Benchmark suite** — New benchmarks: directives, excerpt, phase_breakdown, pipelines, scaling, profile_large_doc. CI threshold checks via `check_thresholds.py`.

- **excerpt.py** — Safer first-line extraction for code blocks; avoids IndexError and unnecessary full-string split.

- **directive.py** — Edge case when `DIRECTIVE_CLOSE` not found: slice `raw_content` to end of source.

- **List parsing** — Avoid duplicate `.split()` calls; use `find("\n")` for line extraction.
