# 0.3.3

**Released**: 2026-03-01

YAML frontmatter parsing with `parse_frontmatter()` and `extract_body()`.

---

## Highlights

- **Frontmatter parsing** — `parse_frontmatter(content)` returns `(metadata, body_markdown)`. Handles valid frontmatter, missing/unclosed delimiters, and invalid YAML with graceful degradation.
- **`extract_body()`** — Strip `---` delimited frontmatter block without YAML parsing. Use when YAML is broken or you only need the body.
- **SSG-friendly** — Normalizes `weight`, `order`, `priority` to float for cascade/sort in static site generators.
- **PyYAML** — Added as core dependency (`pyyaml>=6.0`) for frontmatter support.

---

## Features

### Parse Frontmatter

```python
from patitas import parse_frontmatter, extract_body

content = """---
title: Hello
weight: 10
---
# Body content
"""
metadata, body = parse_frontmatter(content)
# metadata: {"title": "Hello", "weight": 10.0}
# body: "# Body content"
```

### Extract Body (no YAML parse)

When YAML is broken or you only need the body:

```python
from patitas import extract_body

body_only = extract_body(content)
```

### Complementary to Notebooks

- `parse_frontmatter(content)` → `(metadata, body_markdown)`
- `parse_notebook(content)` → `(markdown_content, metadata)`

Same shape for Markdown with frontmatter and Jupyter notebooks.
